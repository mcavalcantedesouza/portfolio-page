## Development Patterns

### Angular Components

Follow standalone component pattern:

```typescript
import { Component } from '@angular/core';

@Component({
  selector: 'app-my-component',
  standalone: true,
  template: `...`,
  styles: [`...`],
})
export class MyComponent {}
```

### Tailwind CSS

Use utility-first approach:

```html
<div class="flex items-center justify-between gap-4 p-4 rounded-lg bg-gray-100">
  <span class="text-sm font-semibold">Title</span>
</div>
```

### Services

Keep business logic separate:

```typescript
@Injectable({ providedIn: 'root' })
export class MyService {
  private http = inject(HttpClient);

  getData() {
    return this.http.get('/api/data');
  }
}
```

# E2E Testing with Playwright - Industry Standard Skill

## Overview

This skill guides you through creating and maintaining end-to-end tests using Playwright following modern industry practices. It includes:

- **Page Object Model (POM)** - Decoupling locators from test logic
- **Fixtures** - Reusable test setup and teardown
- **Step Definition Pattern** - BDD-style test organization
- **Best Practices** - Timeouts, waits, error handling, and reporting

## Project Structure

```
e2e/
├── pages/              # Page Object Models
│   ├── base.page.ts    # Base page with common methods
│   ├── portfolio.page.ts  # Specific page objects
│   └── index.ts        # Export all pages
├── fixtures/           # Custom test fixtures
│   ├── portfolio.fixture.ts  # Portfolio page fixture
│   └── index.ts        # Export all fixtures
├── steps/              # Step definitions (optional BDD pattern)

└── *.spec.ts          # Test files
playwright.config.ts    # Playwright configuration

```

## Key Patterns

### 1. Page Object Model (POM)

Use the Page Object pattern to encapsulate page interactions:

```typescript
// e2e/pages/portfolio.page.ts
export class PortfolioPage extends BasePage {
  readonly headerSelector = 'app-header';

  async isHeaderVisible(): Promise<boolean> {
    return this.isElementVisible(this.headerSelector);
  }
}
```

**Benefits:**

- Locators are centralized
- Tests remain stable when DOM changes
- Reusable methods across multiple tests
- Better maintainability

### 2. Fixtures for Dependency Injection

Use fixtures to inject page objects into tests:

```typescript
// e2e/fixtures/portfolio.fixture.ts
export const test = base.extend<PortfolioFixtures>({
  portfolioPage: async ({ page }, use) => {
    const portfolioPage = new PortfolioPage(page);
    await use(portfolioPage);
  },
});
```

**Benefits:**

- Decoupled test setup
- Automatic cleanup
- Type-safe page objects
- Reusable across tests

### 3. Step-Based Organization

Organize test steps for readability (optional BDD pattern):

```typescript
// e2e/steps/portfolio.steps.ts
export class PortfolioSteps {
  async givenUserIsOnPortfolioPage() {
    await this.portfolioPage.navigateToHome();
  }

  async whenUserScrollsToProjects() {
    await this.portfolioPage.scrollToProjects();
  }

  async thenProjectsSectionIsVisible() {
    await expect(/* ... */);
  }
}
```

## Best Practices

### 1. Use Proper Timeouts

```typescript
// Good - explicit timeout
await page.waitForSelector('.element', { timeout: 10000 });

// Avoid - implicit waits
await page.waitForTimeout(5000); // No, use waitFor methods instead
```

### 2. Leverage Locators API

```typescript
// Good - modern locator API
await page.locator('button:has-text("Submit")').click();
await page.locator('input[type="email"]').fill('test@example.com');

// Acceptable - querySelector
await page.click('button.submit');
```

### 3. Wait for Readiness

```typescript
// Good - wait for application to be ready
await page.waitForLoadState('networkidle');
await page.waitForFunction(() => {
  return document.readyState === 'complete';
});

// Avoid - arbitrary delays
await page.waitForTimeout(5000);
```

### 4. Use Meaningful Assertions

```typescript
// Good - clear intent
await expect(page.locator('button')).toBeVisible();
await expect(page.locator('h1')).toHaveText('Portfolio');
await expect(page.locator('form')).toBeFocused();

// Avoid - vague checks
expect(await page.locator('button').count()).toBeGreaterThan(0);
```

### 5. Handle Dynamic Content

```typescript
// Good - wait for list to load
await page.waitForSelector('app-project-card');
const count = await page.locator('app-project-card').count();

// Avoid - assuming instant availability
const count = await page.locator('app-project-card').count();
```

### 6. Test Organization

```typescript
// Good - descriptive test names and groups
test.describe('Portfolio Page Navigation', () => {
  test('should load all sections', async ({ portfolioPage }) => {
    // ...
  });

  test('should toggle theme successfully', async ({ portfolioPage }) => {
    // ...
  });
});
```

### 7. Error Handling and Debugging

```typescript
// Good - informative error messages
await expect(page.locator('.error-message')).toBeVisible({
  timeout: 5000,
});

// Use --debug flag for interactive debugging
// npx playwright test --debug

// Use trace viewer for failed tests
// npx playwright show-trace trace.zip
```

## Common Test Scenarios

### Navigation Tests

```typescript
test('should navigate through all sections', async ({ portfolioPage }) => {
  await portfolioPage.navigateToHome();
  await expect(portfolioPage.page).toHaveTitle(/Portfolio/);

  await portfolioPage.scrollToAbout();
  await expect(portfolioPage.page.locator('app-about')).toBeVisible();

  await portfolioPage.scrollToProjects();
  await expect(portfolioPage.page.locator('app-projects')).toBeVisible();
});
```

### User Interaction Tests

```typescript
test('should toggle dark mode', async ({ portfolioPage }) => {
  const initialMode = await portfolioPage.page.evaluate(() => {
    return document.documentElement.getAttribute('data-theme');
  });

  await portfolioPage.toggleTheme();

  const newMode = await portfolioPage.page.evaluate(() => {
    return document.documentElement.getAttribute('data-theme');
  });

  expect(newMode).not.toBe(initialMode);
});
```

### API/Network Tests

```typescript
test('should load projects from API', async ({ portfolioPage }) => {
  await portfolioPage.navigateToHome();

  // Wait for API calls
  await portfolioPage.page.waitForLoadState('networkidle');

  const projectsCount = await portfolioPage.getProjectsCount();
  expect(projectsCount).toBeGreaterThan(0);
});
```

### Accessibility Tests

```typescript
test('should have proper heading hierarchy', async ({ portfolioPage }) => {
  await portfolioPage.navigateToHome();

  const h1Count = await portfolioPage.page.locator('h1').count();
  expect(h1Count).toBe(1); // Should have exactly one h1

  const headings = await portfolioPage.page.locator('h1, h2, h3').all();
  expect(headings.length).toBeGreaterThan(0);
});
```

## Troubleshooting

### Test Timeouts

```typescript
// Increase timeout for specific test
test('slow operation', async ({ portfolioPage }) => {
  await portfolioPage.navigateToHome();
  // ...
}, { timeout: 30000 });

// Or set globally in config
use: {
  actionTimeout: 5000,
  navigationTimeout: 30000,
}
```

### Element Not Found

```typescript
// Use better selectors
await page.locator('button:has-text("Click me")').click(); // Good
await page.click('button'); // Ambiguous

// Wait for element
await page.waitForSelector('app-component', { timeout: 10000 });
```

### Flaky Tests

```typescript
// Use proper waits instead of timeouts
await page.waitForLoadState('networkidle');
await page.waitForFunction(() => /* stable state */);

// Use polling assertions
await expect(locator).toHaveCount(1, { timeout: 10000 });
```
